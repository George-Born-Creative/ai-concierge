# StoreKit Configuration — iOS Simulator Testing

`AIConcierge.storekit` is an Xcode StoreKit configuration file. It defines
fake versions of our two App Store subscription products so we can test
Apple In-App Purchase flows in the iOS Simulator **without** needing real
sandbox tester accounts or going through the App Store servers.

## Why it lives here, not in `ios/`

The `ios/` folder is regenerated by `npx expo prebuild --clean` and is
gitignored. Anything we drop in there gets blown away on the next prebuild.
This folder is at the workspace root, outside the ignore rule, so the file
survives prebuilds and stays version-controlled.

## How to wire it up after a fresh prebuild

After `npx expo prebuild` you'll have a freshly generated Xcode workspace
at `ios/AIConcierge.xcworkspace`. Do this once per prebuild:

1. Open the workspace in Xcode (`open ios/AIConcierge.xcworkspace`).
2. In the Project Navigator (left sidebar), drag `ios-storekit/AIConcierge.storekit`
   into the project root. When the dialog appears, choose:
   - **Destination:** "Copy items if needed" — **unchecked** (we want Xcode
     to reference the file in place so edits stay in version control).
   - **Add to targets:** check `AIConcierge`.
3. Edit the run scheme: Product → Scheme → Edit Scheme → **Run** →
   **Options** tab → **StoreKit Configuration** → select
   `AIConcierge.storekit`.
4. Build + run on the Simulator. StoreKit now uses this file instead of
   the live App Store.

## What it contains

Two auto-renewing subscriptions matching what we'll publish in App Store
Connect:

| productID | Display | Price | Period |
| --- | --- | --- | --- |
| `com.daveget.aiconcierge.ghl_pro_monthly` | GoHighLevel Pro | $29.99 | Monthly |
| `com.daveget.aiconcierge.hubspot_pro_monthly` | HubSpot Pro | $29.99 | Monthly |

Both belong to the same subscription group (`AI Concierge Pro`) so the
user can only hold one of them at a time, matching the App Store Connect
configuration described in `.cursor/docs/step-5-app-store-connect.md`.

## Keeping it in sync with App Store Connect

The productIDs in this file **must** match the SKUs you create in App
Store Connect — they're the link between the simulator test environment
and the real one. The price and display name are simulator-only and don't
need to match exactly, but it's tidier if they do.

If you change a productID in App Store Connect, also change it here
**and** in:

- `backend/prisma/seed.ts` — the `appleProductId` seed value
- `components/onboarding/use-apple-iap.native.ts` — `APPLE_PRODUCT_IDS` map

## What you can test in the Simulator with this file

- Plan-selection screen → "Subscribe with Apple" → fake Apple Pay sheet
  appears → confirm → backend `verify` flow runs.
- Restore Purchases → returns the active simulated transaction.
- iOS Settings → Manage Subscriptions → cancel → simulator triggers an
  EXPIRED event you can observe in your Subscription row.

## What you can't test with this file

- Real money. Use sandbox tester accounts on a real device for that.
- App Store Server Notifications V2 — the simulator does **not** call
  out to your `/webhooks/apple` endpoint. For ASN testing you need
  sandbox testers + a real device + the ASN URL configured in App Store
  Connect.

## Tip — force error states

In Xcode → Debug → StoreKit → you can manually fire errors (Verification
Failure, Purchase Failure, etc.) to make sure the app handles each path
gracefully. Worth running through these before the first real submission.
